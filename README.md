# mihomo-gateway

[mihomo](https://github.com/MetaCubeX/mihomo) transparent proxy gateway container with fail-open routing and DNS fallback.

## Variants

| Tag suffix | Platform | CPU target |
|---|---|---|
| `amd64-v1` | linux/amd64 | x86-64 baseline (max compat) |
| `amd64-v2` | linux/amd64 | SSE4.2 |
| `amd64-v3` | linux/amd64 | AVX2 (modern, default for `latest`) |
| `arm64` | linux/arm64 | ARMv8 |

`latest` and version tags map `linux/amd64` → v3, `linux/arm64` → arm64. Pull explicit tags for older CPUs.

## Usage

```yaml
services:
  mihomo:
    image: ghcr.io/<owner>/mihomo-gateway:latest
    container_name: mihomo-gateway
    restart: unless-stopped
    privileged: true
    sysctls:
      net.ipv4.ip_forward: "1"
    volumes:
      - ./config.yaml:/etc/mihomo/config.yaml
    environment:
      MIHOMO_DNS: 127.0.0.1#1053      # mihomo DNS listen address
      FALLBACK_DNS_1: 8.8.8.8          # primary fallback (when mihomo down)
      FALLBACK_DNS_2: 1.1.1.1          # secondary fallback
      WEBUI_ADMIN_PASSWORD: change-me   # required Web UI password
      WEBUI_PORT: 8080                  # optional admin UI port
      # MIHOMO_API_SECRET: change-me-too # optional stable controller secret
    networks:
      macvlan:
        ipv4_address: 192.168.12.200

networks:
  macvlan:
    driver: macvlan
    driver_opts:
      parent: eth0
    ipam:
      config:
        - subnet: 192.168.12.0/23
          gateway: 192.168.12.254
```

The example uses `privileged: true` by default because it is the most compatible option for TUN, routing, and nftables.

To run without privileged mode while still using macvlan, remove `privileged: true` and add:

```yaml
cap_add:
  - NET_ADMIN
  - NET_RAW
devices:
  - /dev/net/tun:/dev/net/tun
```

macvlan gives the container its own LAN IP; it does not grant TUN/routing permissions. With macvlan you do not need `ports:` mappings. Access the admin Web UI at `http://<container-lan-ip>:8080`, and MetaCubeXD at `http://<container-lan-ip>:9090/ui/`.

If you already created a Docker network named `macvlan`, replace the `networks.macvlan` definition with `external: true`.

**Config is required** — mount your mihomo config at `/etc/mihomo/config.yaml`. The container will fail to start without it. See `config-dummy.yaml` in the repo as a reference template.

### Environment variables

| Env | Default | Description |
|---|---|---|
| `MIHOMO_DNS` | `127.0.0.1#1053` | Where dnsmasq forwards DNS for mihomo (must match `dns.listen` in your mihomo config) |
| `FALLBACK_DNS_1` | `8.8.8.8` | Primary fallback DNS when mihomo is down |
| `FALLBACK_DNS_2` | `1.1.1.1` | Secondary fallback DNS when mihomo is down |
| `WEBUI_ADMIN_PASSWORD` | *(required)* | Password for the admin Web UI on `WEBUI_PORT` |
| `WEBUI_PORT` | `8080` | Admin Web UI listen port |
| `MIHOMO_EXTERNAL_CONTROLLER` | `0.0.0.0:9090` | mihomo external-controller bind address and MetaCubeXD backend port |
| `MIHOMO_API_SECRET` | generated at boot | Optional stable secret for mihomo's controller API and MetaCubeXD |

The admin Web UI is protected separately from mihomo's controller API:

- `WEBUI_ADMIN_PASSWORD` signs users into the container control UI.
- `MIHOMO_API_SECRET` is generated by default, passed to mihomo with `-secret`, and prefilled when opening MetaCubeXD from the admin UI. Set it only if you need the same controller secret across container boots.

MetaCubeXD static assets are bundled into the image and served by mihomo at `/ui` on the external-controller port.

The admin Web UI is compiled during the Docker build as a standalone Bun executable. The image builds `bun-linux-x64-baseline-musl` for `linux/amd64` so it remains compatible with the `amd64-v1` tag, and `bun-linux-arm64-musl` for `linux/arm64`.

### mihomo config requirements

Your mounted `config.yaml` must have:
- `dns.listen: 127.0.0.1:1053` (must match `MIHOMO_DNS` env, default `127.0.0.1#1053`)
- `tun.enable: true` with `auto-route: true`

Geo databases (`GeoSite.dat`, `geoip.metadb`, `ASN.mmdb`) are pre-baked into the image.

## Build

```sh
# Local host architecture, loaded into Docker
BUILD_MODE=local ./scripts/build-local.sh

# Single requested platform/variant
BUILD_MODE=single PLATFORM=linux/amd64 VARIANT=v3 ./scripts/build-local.sh

# Release matrix
BUILD_MODE=release ./scripts/build-local.sh
```

`BUILD_MODE=local` detects the host platform (`linux/arm64` on Apple Silicon, `linux/amd64` on x86_64) and uses `--load`.

```sh
# Legacy all-variant local behavior is now:
BUILD_MODE=release IMAGE_NAME=mihomo-gateway ./scripts/build-local.sh
```

```sh
# Direct Docker build example
./scripts/build-local.sh

docker buildx build --load --platform linux/amd64 \
  --build-arg MIHOMO_VARIANT=v3 --tag mihomo-gateway:amd64-v3 .
```

| Build arg | Default | Options |
|---|---|---|
| `MIHOMO_VERSION` | `latest` | `latest` or `v1.19.27` |
| `MIHOMO_VARIANT` | *(empty)* | `v1`/`v2`/`v3` for amd64; empty for arm64 |

Pushes to `main` trigger CI (`.github/workflows/build.yml`) → GHCR.

## Files

```
├── Dockerfile              # Multi-stage: fetch binary + geo DBs, runtime
├── entrypoint.sh           # TUN check, generate dnsmasq.conf, start services
├── nftables.nft            # NAT masquerade rules
├── mihomo.initd            # OpenRC service (supervised, auto-restart)
├── dnsmasq.initd           # OpenRC service (always-on)
├── webui.initd             # OpenRC service for compiled admin UI executable
├── webui/                  # Bun + Elysia Web UI source and unit tests
├── docker-compose.example.yaml  # macvlan examples (new + existing network)
├── .github/workflows/build.yml
└── scripts/build-local.sh
```
